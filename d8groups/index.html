Group Module

  • The Group module allows you to create arbitrary collections of your content and users on your site and grant access control permissions on those collections.
  • Group is very flexible.
  • Everything is an entity.
  • You activate and use only the parts you need.
  • Hooks are available for easy extension.

Advantages of Groups over OG

  • Groups takes full advantage of Entity.
  • Hence all the features of entities also applied for Group.
  • Groups doesn't add custom fields to other entities for the module to work unlike OG module
  • The user experience is enhanced
  • Hooks are available for easy extension.

Disdvantages of Groups compared to OG

  • Groups doesn't allow per group roles/permission.
  • It is not possible to convert existing content into Groups.
  • For each different combination of configurations/permissions we will need to create a new Group Type

Group Types

  • Groups types are like content types but for groups.
  • You can add fields, manage form display and manage displays.
  • For each different combination of configurations/permissions we will need to create a new Group Type

Group Roles and permissions

  • Default Roles comes shipped with groups
  • Anonymous, Outsider and Member
  • There is a settings for adding Admin role
  • Custom Roles can be added as well

Group Roles

  • Anonymous: This is the same as the global Anonymous role, meaning the user has no account.
  • Outsider: This means the user has an account on the site, but is not a member of the group.
  • Member: The default role for anyone in the group. Behaves like the "Authenticated user" role does globally.

Linking Group Types and Content Types

  • You can pick and choose which content types should be enabled in a group type
  • This is a setting at Group Type and not at the group level like in OG
